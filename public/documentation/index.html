Introduction

Event Manager Free is the open-source edition of our event management and ticketing platform built with Laravel. It lets you create and publish single events, sell tickets online via Stripe, manage bookings, and run your site from a modern admin dashboard.

Some admin settings (Classic Light theme, seat plans, recurring/custom schedules, early-bird pricing, add-on services, PDF ticket field customization, attendee forms, and extra payment gateways) are preview-only in this edition. PDF tickets still generate and download with the default layout. Online checkout uses Stripe only. Upgrade to Event Manager Plus to unlock the full product.

Server Requirements

Before installing, ensure your server meets the following requirements. The web installer will also verify these automatically.

Node.js & npmRequired only if you rebuild frontend assets (npm run build)

Required PHP Extensions

  • ctype, fileinfo, json, mbstring, openssl, pdo, tokenizer, xml
  • pdo_mysql (for MySQL/MariaDB database connection)

Writable Directories

  • storage/ and all subdirectories

Production tip: Set APP_DEBUG=false and APP_ENV=production in .env after installation. The web installer does this automatically.

Installation — Web Installer

The recommended way to install Event Manager Free is through the built-in web installer.

  1. Upload files

    Upload all project files to your server. Ensure storage/ and bootstrap/cache/ are writable (chmod 755 or 775).

  2. Install Composer dependencies

    SSH into your server and run:

    cd /path/to/your/project
composer install --no-dev --optimize-autoloader

    For local development you may omit --no-dev.

  3. Create database

    Create an empty MySQL or MariaDB database and note the host, port, database name, username, and password.

  4. Open the installer

    Visit https://your-domain.com/install in your browser. The installer checks PHP version, extensions, and folder permissions.

  5. Configure the application

    On the setup page, enter:

    • Application name and URL (must match exactly how users access the site, including https:// and port if any)
    • Administrator name, email, and password
    • Database connection details (MySQL or MariaDB)

    Optionally check "Erase existing tables" only if you are reinstalling on a database that already has old tables from a previous install.

  6. Complete installation

    Click Run installation. The installer will write .env, run migrations, seed default data, and lock the installer. Save the admin credentials shown on the completion screen.

  7. Sign in

    Go to https://your-domain.com/admin/login and sign in with your administrator account.

Important: Do not refresh or open /install/finish directly. Always submit the form from the setup page. If you see a "Page Expired" (419) error, go back to /install/setup and try again.

Installation — Manual (CLI)

If you prefer command-line installation or the web installer is unavailable:

cd /path/to/your/project
composer install
cp .env.example .env
php artisan key:generate

Edit .env and set at minimum:

APP_NAME="Event Manager Free"
APP_URL=https://your-domain.com
APP_ENV=production
APP_DEBUG=false

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database
DB_USERNAME=your_username
DB_PASSWORD=your_password

ADMIN_NAME="Admin User"
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=your_secure_password

Then run migrations and seeders:

php artisan migrate --force
php artisan db:seed --force
php artisan install:lock

The install:lock command creates the install lock file so the web installer stays disabled on an already-configured site.

Build Frontend Assets (Optional)

Pre-built assets are included. Rebuild only if you modify CSS/JS:

npm install
npm run build

Server Configuration

Apache

Point the document root to the public/ directory. Ensure mod_rewrite is enabled. The included public/.htaccess handles URL rewriting.

If the app is in a subfolder (e.g. http://localhost/EventManagerLaravel/), uncomment and set RewriteBase in the root .htaccess:

# RewriteBase /EventManagerLaravel/

Nginx

server {
    listen 80;
    server_name your-domain.com;
    root /path/to/project/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

APP_URL

APP_URL in .env must match the exact URL visitors use (scheme, host, and port). Payment return URLs and signed links depend on this value.

# Correct examples:
APP_URL=https://yourdomain.com
APP_URL=http://127.0.0.1:8000

Managing Events

Payment Gateways

Configure payments under Admin → Settings → Payment settings. Enable only the gateways you need.

Stripe

  1. Create a Stripe account at stripe.com
  2. Enable Stripe in admin settings and enter your Publishable key and Secret key
  3. For reliable payment confirmation, set up a webhook:
    • Stripe Dashboard → Developers → Webhooks → Add endpoint
    • URL: https://your-domain.com/stripe/webhook
    • Events: checkout.session.completed (and related checkout events as needed)
    • Copy the Signing secret into admin settings

PayPal

  1. Create a PayPal Developer app at developer.paypal.com
  2. Enter Client ID and Secret
  3. Set mode to Sandbox for testing or Live for production
Windows / Laragon: If PayPal returns cURL error 77, download cacert.pem and set PAYPAL_CACERT_PATH in .env to its absolute path.

Razorpay

  1. Create a Razorpay account at razorpay.com
  2. Enter Key ID and Key Secret in admin settings
  3. Enable Razorpay; customers are redirected to a signed payment page

SSLCommerz

  1. Register at sslcommerz.com
  2. Enter Store ID, Store Password, and set Sandbox/Live mode
  3. Configure success, fail, cancel, and IPN URLs in your SSLCommerz merchant panel to match your domain routes

Offline Payments

  • Cash — Customers select cash payment; booking awaits manual confirmation
  • Bank transfer — Display your bank instructions; customers complete transfer offline

Seat Planner

Seat planning is available in Event Manager Plus (premium). In the free edition, seat plan settings are shown as preview-only in the admin panel.

Enable Site-wide

Go to Admin → Settings → Site settings and enable Seat plan. This allows seat planning features across the application.

Enable Per Event

On the event Advanced step, turn on Enable seat plan for this event, then open the seat plan editor (or use Events → Seat plans).

Selected seat numbers appear on bookings, PDF tickets, and check-in screens.

Cron Jobs & Queues

By default, the application uses synchronous queue processing (QUEUE_CONNECTION=sync), which works without a cron job for most sites.

If you enable database queues for high-volume sites, add this cron entry:

* * * * * cd /path/to/project && php artisan schedule:run >> /dev/null 2>&1

And run a queue worker (via Supervisor or similar):

php artisan queue:work --tries=3

After installation, you may optionally switch in .env:

SESSION_DRIVER=database
CACHE_STORE=database
QUEUE_CONNECTION=database

These require that migrations have already been run (the installer handles this).

Troubleshooting

500 Internal Server Error

  • Check storage/logs/laravel.log for details
  • Ensure storage/ and bootstrap/cache/ are writable
  • Run php artisan config:clear and php artisan cache:clear

419 Page Expired (during install)

  • Go to /install/setup and submit the form again (do not refresh /install/finish)
  • Ensure APP_KEY is set in .env

CSS/JS not loading

  • Confirm document root points to public/
  • Run npm run build if assets are missing

Payment redirect fails

  • Verify APP_URL matches your live URL exactly
  • For Stripe, confirm webhook URL and signing secret
  • For PayPal on Windows, set PAYPAL_CACERT_PATH

Images not uploading

  • Check PHP upload_max_filesize and post_max_size
  • Ensure public/uploads/ is writable

Re-run installer

Delete the install lock file at storage/app/install.lock (path may vary) to access /install again. Only do this on a fresh or test environment.

FAQ

Can I use SQLite?

The web installer requires MySQL or MariaDB. For development, you may configure SQLite manually in .env and run migrations via CLI.

How do I change the admin password?

Sign in to the admin panel and update your profile, or change the user record directly in the database.

Where are uploaded files stored?

Uploaded images are stored under public/uploads/ and referenced from the database.

Is the installer accessible after setup?

No. Once installation completes, an install lock file is created and the installer routes are blocked. Use php artisan install:lock for manual deployments.

Can I translate the application?

Language files are in lang/. The default locale is English (en). Set APP_LOCALE in .env and add translation files as needed.
